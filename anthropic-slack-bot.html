毎日自動で新着を検出し、Geminiで日本語要約してSlackに投稿します。課金は新しい記事があり、実際にAPIを叩いたときに発生しやすい構成です。

処理の流れ（あわせて読む用）

  1. 定時起動 — GitHub Actions が main.py を実行する。
  2. 取得 — エンジニアリングブログから記事本文を取り込む。
  3. 重複排除 — processed.json と照合し、未処理のURLだけ進める。
  4. 要約 — Gemini に渡し、指定スキーマのJSON（要約・用語・着想など）を受け取る。
  5. 投稿 — Slack向けに体裁を整え、フリートーク等のチャンネルへ送る。
